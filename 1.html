Komponenten von Java

Java Runtime Environment (JRE)

  • Wird benötigt, um Java-Programme auszuführen
  • Beinhaltet die Java Virtual Machine (JVM)

Java Development Kit (JDK)

  • Beinhaltet Klassen und Tools, um Java zu entwickeln
  • Beinhaltet auch das JRE

Beides ist verfügbar als Oracle Java und OpenJDK.

Im Rahmen dieser Vorlesung wird das JDK benötigt, damit wir Java-Anwendungen entwickeln, kompilieren und starten können. Dazu kann entweder die offizielle Java-Version von Oracle (früher von Sun) oder alternativ die Open-Source-Variante OpenJDK verwendet werden.

Zwei Tools: javac und java

1.6

javac - Der Java Compiler

  • Eingabe: *.java-Dateien (Klassen-Quellcode)
  • Ausgabe: *.class-Dateien (plattformunabh. Bytecode)

java - Starten einer Java-Anwendung

  • Eingabe: Name einer Klasse
  • Startet main()-Methode in der gegebenen Klasse

          $ javac MeinProgramm.java
          $ ls
          MeinProgramm.class	MeinProgramm.java
          $ java MeinProgramm
          Hallo!
Der Java-Compiler javac erzeugt keinen direkt ausführbaren Maschinencode, sondern einen Bytecode, der in der JVM (Java Virtual Machine) unter Windows, Linux, MacOS etc. ausgeführt werden kann.

Beispiel-Programm

MeinProgramm.java

Wichtig: Der Dateiname muss gleich dem Klassennamen sein!

Eine Klasse, welche man starten möchte, benötigt die Methode public static void main(String[] args). Die Kommandos, die dort drinstehen, werden beim Programmstart ausgeführt. In IntelliJ kann man die main-Methode mit Hilfe eines Live-Templates schnell generieren lassen, indem man einfach main eingibt und die Tabulatortaste drückt.
Im gezeigten Beispiel gibt das Programm einfach nur den Text "Hallo!" auf der Konsole aus. Unsere Klasse heißt MeinProgamm, daher muss der Dateiname unbedingt MeinProgramm.java heißen, auch Groß-Kleinschreibung ist hier wichtig. Dies bedeutet daher auch, dass es innerhalb einer .java-Datei nur eine Klasse geben darf, die public ist. public bedeutet, dass sie von außen sichtbar ist.

Variablen

3

Variablen speichern einen Wert eines primitiven Datentypen oder die Referenz auf ein Objekt.

Jedes Java-Kommando muss mit einem Semikolon ; enden!

Variablen müssen zuerst deklariert werden, bevor man sie verwenden kann. Deklarieren erfolgt entweder mittels datentyp variablenname; oder var variablenname = ...; Bei letzterer Variante ermittelt der Java-Compiler den passenden Datentyp. Im oben stehenden Beispiel wird c=5 gesetzt, außerdem wird auch der neue Wert (also 5) zurückgegeben. Somit hat auch b den Wert 5, usw. Schlussendlich setzt also a=b=c=5; alle drei Variablen auf 5.

Konstanten

3.6

final macht Variablen unveränderbar.

ANZAHL_SPIELER = 3; ← verboten!

Der Wert einer Konstanten kann nur einmalig gesetzt und danach nicht mehr verändert werden. Wird beim Deklarieren der Variable ihr Wert nicht initial gesetzt (wie im API-KEY-Beispiel), darf dies nachfolgend genau einmal gemacht werden. Die Konstante PETER ist ein Objekt der Klasse Person. final bedeutet, dass die Variable niemals ein anderes Objekt referenzieren darf. Es ist jedoch erlaubt, dass das referenzierte Objekt modifiziert wird.

switch

5.3
switch wertet einen Ausdruck aus (hier die Variable sprache) und überprüft, welchen Wert dieser hat. Dementsprechend werden diejenigen Anweisungen, die hinter dem jeweiligen Doppelpunkt stehen, ausgeführt. Wichtig ist, dass die letzte Anweisung ein break; ist, damit die anderen Fälle nicht weiter überprüft werden müssen. Der default-Fall ist optional. Die dahinter stehenden Ausweisungen werden ausgeführt, wenn vorher nicht bereits der Switch-Block mit einem break verlassen wurde, also wenn im gezeigten Beispiel die Sprache weder Englisch noch Deutsch ist.

while / do-while / for

5.6, 5.4
Der Unterschied zwischen der while-Schleife und der do-while-Schleife ist, dass bei letzter in jedem Falle der Schleifenkörper einmal ausgeführt wird, erst danach wird überprüft, ob weitere Durchläufe erfolgen sollen.

Strings

7

Strings stehen in doppelten Anführungszeichen.

Sobald bei Verwendung des +-Operators einer der Operanden ein String ist, werden die anderen Operanden in Strings umgewandelt.

String.equals

Strings in Java sind immutable, das heißt ein String kann nicht verändert werden. Stattdessen wird ein neues String-Objekt erzeugt und seine Variable zeigt nun auf den neuen String.

NICHT (!!!) if(name == "Herr Peter Müller")

Würde man zwei Strings mit == vergleichen, ist das Ergebnis fast immer false. Verglichen wird dann nämlich nicht, ob die Strings den gleichen Text beinhalten, sondern ob es sich um die gleichen String-Objekte im Arbeitsspeicher handelt. Und das ist in der Regel nicht der Fall.
Viele Entwickler schreiben auch if("Herr Peter Müller".equals(name)) statt der auf dieser Folie stehenden Form name.equals. Der Vorteil daran ist, dass dies auch dann funktioniert wenn die Variable name den Wert null beinhaltet. Dann würde der Vergleich false zurückliefern, während name.equals eine NullPointException werfen würde.

Arrays

6
Die Array-Variable erhält den Typ datentyp[]. Danach kann der Array entweder mittels new datentyp[n] initialisiert werden (n ist die Anzahl der im Array zu speichernden Elemente) oder über ein direktes Zuweisen der Elemente mittels der {}-Syntax. Die Länge des Arrays ist immer fix, sie kann nachträglich nicht mehr verändert werden. Der Zugriff auf die einzelnen Array-Elemente erfolgt mittels [i], wobei der Index i eine ganze Zahl zwischen 0 und n-1 ist. Greift man auf ein ungültiges Element zu (z. B. tage[7]), erscheint eine ArrayIndexOutOfBoundsException.

for-each-Schleife

Die for-each-Schleife ermöglicht ein simples Iterieren über die Elemente eines Arrays. Der Wert des aktuellen Elements wird in jeder Iteration der vor dem Doppelpunkt stehenden Variable zugewiesen. Die for-each-Schleife kann die Elemente des Arrays jedoch nicht verändern, d. h. mit zahl = 0 kann man nicht jede einzelne Zahl im lotto-Array auf 0 setzen.

Textausgabe: System.out

3.5
java.lang.System muss nicht importiert werden, da es sich im Paket java.lang befindet. System.out ist ein sogenannter PrintStream, welcher Methoden anbietet, mit denen Text auf der Konsole ausgegeben werden kann. print macht anders als println am Ende keinen Zeilenumbruch. Diesen kann man aber mit \n innerhalb des Strings auch manuell machen. Die Methode printf nimmt als ersten Parameter einen String, der ausgegeben werden soll und in dem Platzhalter vorkommen. Als weitere Parameter wird für jeden Platzhalter ein Wert gesetzt. Der Platzhalter %s steht für String, %d für eine ganze Zahl im Dezimalsystem, %.4f für eine auf vier Nachkommastellen gerundete Dezimalzahl.

Texteingabe: Scanner

3.5
Die Klasse Scanner befindet sich im Paket java.util, dahier muss diese Klasse zunächst importiert werden. Nach der Erzeugung eines Scanner-Objekts scan kann mittels scan.nextLine() die vom Benutzer eingegebene Zeichenkette gelesen werden. Im gezeigten Beispiel wird dies in einer Dauerschleife gemacht. Erst wenn die Eingabe leer ist, wird die Schleife durch das break;-Kommando verlassen. Der Dialog in diesem Programm könnte wie folgt aussehen: "Was sagst du?" - "Hallo" - "Ich schreie zurück: HALLO" - "Was sagst du?" - "tschö" - "Ich schreie zurück: TSCHÖ" - "Was sagst du?" - ""
