Aufgabe 1: JavaFX, Threads

Abgabe-Deadline: 08.07.2020 12:00 im Moodle

Programmieren Sie das folgende Spiel "Tipptrainer" mit JavaFX:

Ziel des Tipptrainers ist es, seine Fähigkeiten im 10-Finger-Schreiben auf der Computertastatur zu verbessern. Der Benutzer hat 15 Sekunden Zeit, einen Text zu verfassen, daraufhin wird die Anzahl der Anschläge pro Minute angezeigt.

Die Anwendung besteht aus einem mehrzeiligen Eingabefeld, in welches der Benutzer einen Text schreiben kann, einem Label, welches die Anchläge pro Minute darstellt und einem Neustart-Button.

Sobald man anfängt zu tippen, läuft im Hintergrund die Zeit. Es ist nicht nötig, dass die verbleibende Zeit irgendwo angezeigt wird. Aber wenn 15 Sekunden abgelaufen sind, deaktiviert sich das Eingabefeld (disable), sodass ein Weitertippen nicht mehr möglich ist. Das Label zeigt nun die erreichte Punktzahl an, also die Anzahl der Zeichen pro Minute.

Ein Klick auf den Neustart-Button leert und aktiviert das Tippfeld wieder, sodass man erneut spielen kann.

Hinweise:

  • Verwenden Sie FXML, um das Layout zu erstellen und entwickeln Sie eine Controller-Klasse.
  • Es ist nicht möglich, dass eigens erstellte Threads auf direkte Weise auf die JavaFX-Elemente zugreifen. Versuchen Sie dies, sehen Sie eine Exception "Not on FX application thread".
  • Hier ist die Lösung, um dennoch in Ihren Threads auf JavaFX-Elemente zugreifen zu können: Rufen Sie die entsprechenden Kommandos in einem solchen Block innerhalb Ihres Threads auf: 
    Platform.runLater(() -> { 
    // hier
});

Laden Sie im Moodle einen Screenshot Ihrer Anwendung (mit Anzeige der Anschläge pro Minute!) hoch und zeigen Sie in Ihrer Einzelslot-Session Ihren Code und die Anwendung.

Optional:

  • Darstellung der Restlaufzeit in Sekunden.
  • Wird der Neustart-Button geklickt, während die Zeit noch läuft, soll der Countdown abgebrochen werden.
