Als weiteres benötigen wir eine Möglichkeit, JSON-Daten zu parsen, damit wir später in der Witze-App auf das Ergebnis eines API-Aufrufs zugreifen können. Die APIs, die wir verwenden, liefern als Rückgabe einen JSON-String.

JSON-Objekte sehen in etwa so aus: {"attribut": "wert"}. Während der Attributname stets ein String ist, kann der Wert ein "String", eine Zahl, ein [Array], ein {Unterobjekt}, usw. sein.

Diesem Projekt wurde bereits die org.json-Bibliothek hinzugefügt. Mit ihr kann man in Java JSON-Daten verarbeiten. Das Hinzufügen von Bibliotheken behandeln wir im Detail jedoch erst später.

Das Parsen eines JSON-Strings erfolgt in der Methode APIHelper.parseJson. Dort kann einiges schief gehen: Wenn beispielswiese der übergebene String kein wohlgeformtes JSON ist, wird eine JSONException geworfen. Dies ist eine Subklasse von RuntimeException.

Fangen Sie in der Methode parseJson die JSONException ab und werfen Sie eine new IllegalArgumentException("Ungueltiges JSON-Objekt").

Die IllegalArgumentsException ist ebenfalls eine Subklasse von RuntimeException. Sie muss also nicht abgefangen werden und bedarf auch einer throws-Deklaration.

Probieren Sie im Anschluss in der Main-Klasse aus, was passiert, wenn Sie einen nicht-wohlgeformten JSON-String übergeben (vergessen Sie z. B. ein " oder eine }) oder wenn Sie einen Array [1,3,3] übergeben.
